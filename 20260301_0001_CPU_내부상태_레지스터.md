1. x86 CPU 레지스터  
CPU 에는 레지스터가 있다.
일반 목적의 레지스터가 있고(흔히 말하는 범용 레지스터)
특수 목적의 레지스터가 있다.(내부상태 레지스터등)

일반목적의 레지스터는 AX, BX, CX, DX 등의 레지스터가 있고,
SP - 스텍포인터 (스텍의 끝)
BP - 베이스포인터 (스텍의 시작)
IP - 인스트렉션 포인터(흔히 말하는 프로그램 카운터, 현재 실행 코드를 가리킴)
FLAGS - 플래그 레지스터(내부 상태 정보 적재 - 뒤에서 정리)
CS(코드세그먼트), DS(데이터세그먼트), SS(스텍세그먼트) - 세그먼트 주소 적재 레지스터


2. 내부 상태 레지스터(FLAGS)
CPU 명령처리시 ADD, SUB 등의 명령은 덧셈, 뺄셈으로서 
그 결과 자체는 특정 데이터가 되지만,
보통은 프로그램 특성상 해당 데이터를 가지고 조건처리등을 통해 분기를 한다.

CPU 는 복잡한 작업을 소수의 명령어로 반복처리 하는 형태 이다.
그런데, 분기처리시 매번 데이터를 가져와서 해석하는 형태로 하게 되면, CPU 입장에는 비효율적이다.
최대한 작업을 압축해서 해야 하나, 매번 반복처리를 하는 것은 비효울 적이다.
그런데, 데이터 형태는 다양하므로, 분기처리등의 작업을 위해 매번 반복하지 않고,
해당 처리 결과 상태를 별도의 저장소(레지스터)에 담아두고,
해당 레지스터 값을 참고하여 분기처리를 한다.

그 역할을 내부상태 레지스터라고 한다.
8088 에는 FLAGS 레지스터가 있다. 각 1비트씩 특정 index 에  
ZF(zero flag) - 0 인지 
SF(sign flag) - signed 인지 (MSB 값을 그대로 옮김)
CF(carry flag) - 연산결과 저장공간 자리수를 벗어나는지 
OF(overflow flag) - signed 로 판단시 부호가 바뀌는지 
등의 레지스터를 뒀고,
이후에 세대를 증가하면서 스펙을 늘려갔다.

80386 부터는 EFLAGS 
x86_64(AMD64) 에서는 RFLAGS
 
호환성을 위해 사이즈가 커지지만, 미리 사용중인 index 를 유지함에 따라 호환성을 유지한다.

내부상태 레지스터를 사용함으로서,
해당 레지스터 값을 활용하여, 조건처리(분기처리)등 에서 효율적인 처리로 성능 향상을 가능케 한다.
